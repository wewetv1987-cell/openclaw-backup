### 学习内容

#### 1. 容器基础概念
- 容器定义: 隔离的应用进程
- 核心特性: 自包含、隔离、独立、可移植
- 与 VM 对比: 共享内核，更轻量

#### 2. 镜像原理
- 不可变性: 创建后不可修改
- 分层结构: 每层代表文件系统变更
- 镜像来源: Docker Hub 官方镜像、硬ening镜像、验证发布者

#### 3. Docker Compose
- 声明式配置: YAML 文件定义多容器应用
- 单文件管理: compose.yaml 包含所有服务
- 一键部署/清理: up/down 命令

#### 4. 最佳实践
- 单一职责原则: 每容器一个服务
- 分层优化: 利用缓存加速构建
- 数据持久化: 使用 Volumes
- 网络管理: Compose 自动创建网络

#### 5. 常用命令
- 容器: run, ps, stop, rm
- 镜像: pull, ls, history, rm
- Compose: up, down, logs
